# MineLua 🟩
### Minecraft Bedrock Edition Server — Powered by Lua

MineLua is a fully featured Minecraft Bedrock Edition compatible server written in Lua.  
It reads real world folders, supports `.plug` plugins (Lua + textures + data inside a zip), and runs on Koyeb via Docker.

---

## Features

| Feature | Details |
|---|---|
| **Protocol** | MCBE 1.0 – 1.21.x (60+ protocol versions) |
| **Blocks** | All 700+ MCBE blocks |
| **Items** | All 700+ MCBE items |
| **Mobs** | All 100+ MCBE mobs (registry + spawning hooks) |
| **World loading** | Reads real MCBE world folders (LevelDB + custom format) |
| **World generation** | Procedural Perlin-noise terrain, biomes, caves, ores, trees |
| **Plugins** | `.plug` files (zip containing `main.lua` + `plugin.yml` + textures) |

| **Scheduler** | Tick-based delayed and repeating tasks |
| **Inventory** | Full inventory, give, clear, drop |
| **Forms** | Modal / simple / custom form support |
| **Deployment** | Docker + Koyeb ready |

## Quick Start

### Requirements
- Lua 5.4
- LuaRocks packages: `luasocket`, `lua-zlib`, `luazip`

### Run locally
```bash
# Install deps (Ubuntu/Debian)
sudo apt install lua5.4 luarocks
sudo luarocks install luasocket
sudo luarocks install lua-zlib
sudo luarocks install luazip

# Clone and run
git clone https://github.com/your-user/MineLua
cd MineLua
lua5.4 src/main.lua
```

### Run with Docker
```bash
docker build -t minelua .
docker run -d \
  -p 19132:19132/udp \
  -v $(pwd)/worlds:/minelua/worlds \
  -v $(pwd)/plugins:/minelua/plugins \
  --name minelua \
  minelua
```

### Deploy to Koyeb
1. Push image to Docker Hub:
   ```bash
   docker tag minelua your-user/minelua:latest
   docker push your-user/minelua:latest
   ```
2. Edit `koyeb.yaml` with your image name.
3. Deploy:
   ```bash
   koyeb app init minelua --manifest koyeb.yaml
   ```

## Worlds

MineLua reads worlds from the `worlds/` directory.  
Each subfolder is a world. It supports:

- **MCBE LevelDB format** — real worlds exported from Minecraft (Bedrock)
- **Custom binary format** — generated by MineLua itself
- **Procedural generation** — if no world data is found, terrain is generated

```
worlds/
  world/          ← default world
    level.dat     ← MCBE level data (seed, spawn, etc.)
    db/           ← LevelDB chunk storage
    players/      ← Per-player data files
```

To use a real Bedrock world: copy the world folder from your device into `worlds/`.

## Plugins (`.plug` files)

A `.plug` file is a **ZIP archive** with this structure:

```
MyPlugin.plug  (zip)
├── plugin.yml         ← metadata (required)
├── main.lua           ← entry point (required)
├── lib/               ← additional Lua modules
│   └── utils.lua
├── textures/          ← optional resource textures
│   └── custom_block.png
└── data/              ← optional data files
    └── config.json
```

### `plugin.yml`
```yaml
name: MyPlugin
version: 1.0.0
author: YourName
description: A cool plugin
main: main.lua
api: "1.0"
```

### `main.lua` — Plugin API

```lua
-- Called when plugin loads
function onLoad()
    Logger.info("Plugin loaded!")

    -- Listen for events
    Server.registerEvent("PlayerJoin", function(event)
        event.player:sendMessage("§aWelcome, " .. event.player.name .. "!")
    end)

    -- Register a command
    Server.registerCommand("hello", function(player, args)
        player:sendMessage("Hello!")
    end, "Say hello")

    -- Schedule a repeating task
    Server.scheduleRepeating(0, 200, function()
        Server.broadcastMessage("§7Tick...")
    end)
end

-- Called when plugin unloads
function onUnload()
    Logger.info("Plugin unloaded!")
end
```

### Full Server API

| Method | Description |
|---|---|
| `Server.getPlayer(name)` | Get player by name |
| `Server.getOnlinePlayers()` | All online players |
| `Server.broadcastMessage(msg)` | Broadcast to everyone |
| `Server.getDefaultWorld()` | Get default world |
| `Server.getWorld(name)` | Get world by name |
| `Server.registerEvent(name, cb)` | Listen to an event |
| `Server.fireEvent(name, data)` | Fire a custom event |
| `Server.registerCommand(name, fn, desc)` | Add a command |
| `Server.scheduleTask(ticks, fn)` | Run once after delay |
| `Server.scheduleRepeating(delay, interval, fn)` | Run repeatedly |
| `Server.cancelTask(id)` | Cancel a task |
| `Server.getConfig()` | Get server config |
| `Server.getVersion()` | Server version string |
